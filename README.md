# Лендинги Мегафон

Репозиторий для хранения и визуального тестирования лендингов через Vercel.

## Структура репозитория

```
/
  megafon/
    prod/
      eroportal/
        eroportal_41/
          index.html
          index_files/
            style.css
            play.png
      inero/
        inero_61/
      videomir/
        videomir_104/
    test/
      eroportal/
        ep35-01/
      inero/
        ie35-20/
      videomir/
        vm35-01/
  vercel.json
  index.html
```

## Как посмотреть лендинг по ссылке

```
kasper-ten-gray.vercel.app/megafon/prod/inero/inero_61/
```

Слэш в конце обязателен, иначе CSS не подгрузится.

## Как добавить новый лендинг

1. Создай папку в `test` или `prod` — номер следующий за последним
2. Положи файлы внутрь:

```
inero_62/
  index.html
  index_files/
    style.css
    play.png
```

3. Запушь:

```
git add .
git commit -m "добавил inero_62"
git push
```

## Workflow

1. Создаёшь лендинг в `test` — смотришь по ссылке, итерируешь
2. Когда утверждён — копируешь в `prod`
3. Боевую версию из `prod` заливаешь во фронт-офис Мегафона

## Именование папок

| Сервис    | Пример         |
| --------- | -------------- |
| ИнЭро     | `inero_61`     |
| Эропортал | `eroportal_42` |
| Видеомир  | `videomir_105` |

## offer_link

- **test** — `{offer_link}` остаётся на кнопке, фронт-офис подставит ссылку при тестировании
- **prod** — `{offer_link}` убирается, кнопка работает через JS (`onclick="showInfo()"`)

## Плейсхолдеры

Следующие плейсхолдеры не трогать — фронт-офис подставит их автоматически:

- `SUBNAME` — название сервиса
- `SUBPRICE` — цена подписки
- `PROVIDER` — контент-провайдер

## Правила

- Всегда делай `git pull` перед началом работы
- CSS и картинки всегда в папке `index_files` рядом с `index.html`
- Не удалять боевые лендинги из `prod` без согласования

## Основные команды

```
git pull          — получить изменения коллеги
git add .         — добавить все изменения
git commit -m ""  — сделать коммит
git push          — запушить на GitHub
```
